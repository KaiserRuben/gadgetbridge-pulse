## 1. Landing — What You See When You Open the App

**Default route: `/today`**

Decision rationale: the user described wanting a "better Huawei Health." Huawei Health opens to a daily summary. The user is a data person who reaches for `/explore` during deliberate sessions, not glances. `/today` is the glance; `/explore` is the session. Making `/explore` the default would punish every casual open with a chart-grid that requires orientation. The LLM article on `/today` is the load-bearing daily narrative; everything else radiates from it.

The existing `/today` page (RSC, `today/page.tsx`) already does this well. What is missing: a persistent date strip at the top, "Why?" affordance on anomalies, and a link footer to `/explore` for the same date.

## 3. Navigation Primitives

### Persistent nav

**Mobile**: floating pill bottom nav (`components/layout/bottom-nav.tsx`). Current items: Heute, Woche, Alarme + sheet (Profil, Labs). Add **Explore** (BarChart2 icon) as fourth primary slot; demote Woche to the sheet. Four primary + sheet is the right density for this user's actual usage pattern (today + explore are daily; week is weekly; alarms/profile/labs are occasional).

New primary slots: Heute · Explore · Alarme · (sheet: Woche, Profil, Labs, Log)

The existing `v2-nav-pill` `layoutId` spring handles the pill transition across all four without change.

**Desktop**: sidebar (`components/layout/desktop-nav.tsx`). Add Explore above the divider. Add Log below the divider alongside Profil and Labs.

### Date navigation — three levels

**Level 1 — DateStrip**: A sticky 14-day horizontal scroll strip rendered at the top of `/today`, `/today/[domain]`, and `/explore`. Each cell: day abbrev + date number + small verdict-band dot (same `bandColor` logic as `WeekStrip`). Active cell has a `layoutId="date-strip-cursor"` motion pill. Tap any cell → `router.push` with `?date=` param; the `PageTransition` directional slide (already exists in `components/motion/page-transition.tsx`) fires left or right based on date order.

**Level 2 — Calendar drawer**: A `<Dialog>` sheet (same Radix Dialog pattern as `MoreSheet` in `bottom-nav.tsx`) containing a month calendar grid. Cells with data have their verdict-band dot. Tapping a cell commits `?date=` and closes. Reachable from a calendar icon at the right end of the DateStrip. On desktop this opens as a popover instead.

**Level 3 — Year heatmap nav** (`/explore/heatmap`): Clicking any cell navigates to `/today?date=YYYY-MM-DD`. This is the GitHub-grid UX research item — the heatmap is itself navigable, not just decorative.

### Drill-down breadcrumb pattern

Every drill-down route renders a breadcrumb link back to its parent in the header, following the existing pattern in `today/[domain]/page.tsx` (ArrowLeft + "zurück zu Heute"). Extend this to:

- `/today/[domain]` → `← Heute` (existing)
- `/explore/[metric]` → `← Explore`
- `/explore/[metric]?date=X` → `← Explore · [date]`
- `/explore/heatmap` → `← Explore`
- `/explore/correlate` → `← Explore`
- `/explore/patterns` → `← Explore`

The breadcrumb uses the same monospace muted style at `text-[13px] font-mono` already in the domain page.

### Link patterns

- `<Link prefetch={false}>` for all drill-downs (existing pattern, prevents eager data prefetch on chart-heavy pages).
- `router.push` (CSR) for date strip navigation to avoid full-page RSC rerender when only `?date=` changes.
- `<Link>` for nav items (existing).

## 4. Drill-Down Architecture

### Hierarchy

```
/today
  └─ /today/[domain]?date=         shared-layout morph (DriverChip → HeroMorph)
       └─ chart row click → inline expanded sample window (FLIP, same page)
            └─ "raw samples" link → /explore/[metric]?date=  (route push)

/explore
  └─ KPI tile click → /explore/[metric]        shared-layout (tile → page hero)
       └─ chart zoom selection → inline date-range lock (CSR state, no route)
            └─ data point click → inline sample row expand (FLIP)
       └─ /explore/[metric]?date= → day anchored view
  └─ /explore/heatmap
       └─ cell click → /today?date=  (route push)
  └─ /explore/correlate → no further drill
  └─ /explore/patterns
       └─ pattern card click → /explore/[metric]?pattern=id  (route push + highlight)
```

### Shared-layout transitions (motion/react `layoutId`)

1. **DriverChip → HeroMorph** (existing): `layoutId="driver-{metricId}"` on the chip in `/today`, same id on `<HeroMorph>` in `/today/[domain]`. This is already implemented and ships.

2. **KPI tile → /explore/[metric] hero**: `layoutId="explore-tile-{metric}"` on the KPI tile value span; same id on the large metric display at the top of `/explore/[metric]`. New, to implement.

3. **Date strip cursor pill**: `layoutId="date-strip-cursor"` on the active cell background, slides between cells on date change. New.

4. **Nav pill**: `layoutId="v2-nav-pill"` (existing, already ships across both nav components).

### Route-push transitions (PageTransition)

`PageTransition` in `components/motion/page-transition.tsx` already does directional slide (±16 px, 240 ms) based on nav index. Extend `navIndex` to include explore routes so `/explore` → `/explore/[metric]` slides right (deeper), back slides left.

### "Any chart should be drillable to raw samples"

Every chart card in `/today/[domain]` and `/explore/[metric]` renders a small `[raw data ↗]` link in the `CardHeader hint` slot that navigates to `/explore/[metric]?date=YYYY-MM-DD`. This is a static `<Link>`, no animation, because the destination is a full chart page not a panel.

For within-page expansion (sample window): clicking a data point on a Recharts chart fires an `onDatapointClick` callback. The chart component passes the selected index up; the page renders an expanded `<SampleRow>` below the chart using `motion.div` with FLIP (`layout` prop). Reduced-motion: no layout animation, row appears instantly.

### "Why?" button on anomalies — inline expansion, not modal

`AnomalyRow` (`components/ui/anomaly-row.tsx`) gains an optional `onWhyClick` / `whyContent` prop pair. When `whyContent` is null, a "Warum?" button appears. Clicking fires the LLM anomaly-explanation call (PROBE_anomaly_explanation.md: Probe 2 scope, 7-day `_facts.json`, ~26 s). While loading, the row expands to show a skeleton. On resolution, the hypotheses list renders inline using `motion.div layout` — the row grows, pushing content below via FLIP. No modal, no new route.

The anomaly explanation LLM call (PROBE result: GO, Probe 2 scope) is triggered on demand from the client. It is a `fetch` to a route handler `/api/explain-anomaly?id=X&date=Y`. The route handler does the Ollama call. The expansion is CSR state, not persisted.

The "Why?" pattern propagates to: `/today` anomaly rows, `/today/[domain]` driver chips (on tap/click), `/explore/[metric]` anomaly callouts on the timeline.

## 5. Animation as Wayfinding

The existing motion tokens live in `/lib/motion.ts` (`SPRING_VERDICT` is referenced in `hero-morph.tsx`). The existing transitions are:

- **FadeUp** (`components/motion/fade-up.tsx`): `duration: 0.4`, `ease: [0.22, 1, 0.36, 1]`, `y: 10`. Used on page load stagger.
- **ChartReveal** (`components/motion/chart-reveal.tsx`): clip-path wipe left→right, `duration: 0.8`, same ease. Used on scroll-into-view.
- **HeroMorph** (`components/motion/hero-morph.tsx`): shared-layout spring via `SPRING_VERDICT`.
- **PageTransition** (`components/motion/page-transition.tsx`): `opacity` + `x: ±16`, `duration: 0.24`.
- **Nav pill**: spring `stiffness: 400, damping: 32`.

New transitions to add (all respect `useReducedMotion`):

### T1 — DateCursor slide
- **What**: shared-layout pill sliding between date strip cells.
- **Spec**: `layoutId="date-strip-cursor"`, spring `stiffness: 500, damping: 38`. Faster than nav pill to feel snappy on rapid taps.
- **Fallback**: instant background color swap.

### T2 — TileHero morph
- **What**: KPI tile number morphs to the large hero number on `/explore/[metric]`.
- **Spec**: same spring as `SPRING_VERDICT` (`stiffness: 300, damping: 30`). `layoutId="explore-tile-{metric}"`.
- **Fallback**: FadeUp on the hero instead.

### T3 — InlineExpand (FLIP for "Why?" and sample rows)
- **What**: anomaly row or sample row growing in place; siblings pushed down.
- **Spec**: `motion.div` with `layout` prop on the growing element, `transition={{ layout: { duration: 0.3, ease: [0.22, 1, 0.36, 1] } }}`. The container (`flex flex-col gap-3`) is also wrapped in `motion.div` with `layout`.
- **Fallback**: no `layout` prop; content appears without row push.

### T4 — DrawerSlide
- **What**: calendar drawer and "More" sheet sliding up from bottom.
- **Spec**: existing pattern from `MoreSheet` (`y: 40 → 0, spring stiffness: 360, damping: 30`). Reuse for calendar drawer.
- **Fallback**: Radix Dialog default (opacity only).

### T5 — HeatmapCellExpand
- **What**: hovering a heatmap cell produces a small tooltip that scales up from the cell. On click, the cell pulses before navigation.
- **Spec**: tooltip `scale: 0.8 → 1, opacity: 0 → 1, duration: 0.18`. Click pulse: `scale: 1 → 0.9 → 1, duration: 0.2`.
- **Fallback**: static tooltip, no pulse.

### T6 — RouteBackdrop crossfade (existing, extend)
- Already ships in `components/layout/route-backdrop.tsx`: two radial blobs, `duration: 0.6, ease: [0.22, 1, 0.36, 1]`.
- Extend `ROUTE_ACCENT` to cover `/explore` (activity blue → hrv teal) and `/log` (stress amber → activity green).

### T7 — NumberTicker on KPI grid
- `components/ui/number-ticker.tsx` already exists and is used in `KpiTile`. Fire it on mount for the `/explore` KPI grid with stagger via `Stagger` + `StaggerItem` (already in `fade-up.tsx`), `step: 0.04` for 12 tiles.

### T8 — ChartZoomSelect
- **What**: drag-selecting a time range on a Recharts chart produces a visible selection rectangle, then snaps to the selected window.
- **Spec**: `ReferenceArea` with animated opacity `0 → 0.15` on drag, same `ease` curve. Not a motion/react animation — Recharts `ReferenceArea` + CSS transition `opacity 150ms`.
- **Fallback**: static ReferenceArea, no fade.

**Timing philosophy**: all eases use `[0.22, 1, 0.36, 1]` (existing standard). Springs use the existing `SPRING_VERDICT` or the faster `stiffness: 400-500` for UI elements (nav, date cursor). Duration floor: 150 ms. Duration cap: 800 ms (ChartReveal). No bounce.

## 6. `/today` Architecture

The existing article structure stays. Changes only:

**Add: DateStrip at top**
A sticky `<DateStrip>` component rendered above the `<article>` in `today/page.tsx`. It is a client component receiving `activeDate` and navigating via `router.push`. The 14-day window is computed server-side and passed as a prop array (date + verdict_band) so the strip renders with correct colors SSR. On mobile it scrolls horizontally.

**Add: "Why?" per anomaly**
`AnomalyRow` already receives `severity`, `title`, `detail`. Add `observationId` and `date` props. Render a "Warum?" button (small, monospace, muted) inline. On click: expand with InlineExpand (T3), fetch `/api/explain-anomaly`, show skeleton, then render hypothesis list (factor, strength badge, rationale). This maps directly to PROBE_anomaly_explanation.md: Probe 2 scope, schema `{hypotheses: [{factor, strength, rationale}]}`, at most 5 items. Display strength as a `BandBadge`-style chip (strong=up, moderate=steady, weak/unlikely=low-signal).

**Add: CoachCards below anomalies**
One `CoachCard` per active lever (from Phase 3 runner output). The existing `components/coach/coach-card.tsx` is used as-is. The cards show `trajectory` text, the tiny step, and the `projection_90d` text. Per the coaching-trajectory probe the tiny step is cached per `(user, lever_id, iso_week)` to prevent churn. Cards are collapsible — default: trajectory + tiny step visible; projection + confidence behind a `<details>`.

**Add: Explore footer link**
At the bottom of the article: a single `<Link href="/explore?date={dateKey}">` styled as a muted `surface-hover` card with the text "Rohdaten für diesen Tag → Explore" and a BarChart2 icon.

**Remove: reasoning trace `<details>`**
Move to `/labs` or a dev-only flag. It is a data-quality debug tool, not user-facing in production.

## 7. `/explore` Architecture

`/explore` is a new route group: `app/(dashboard)/explore/`.

### `/explore` — metric grid landing

**Layout**: RSC shell with `<DateStrip>` (same component as `/today`). Below: 12 KPI tiles in a responsive grid (`grid-cols-2 sm:grid-cols-3 @[60rem]:grid-cols-4`). Below fold: year heatmap (GitHub-grid style, 53 weeks × 7 days).

**KPI tiles**: use existing `KpiTile` component. Enhance with: sparkline (14-day, using `SparklineInline`), percentile chip (`P72` styled as an `eyebrow` badge), z-score chip (`+2.1σ` styled muted). The `href` prop on `KpiTile` navigates to `/explore/[metric]`. The `layoutId="explore-tile-{metric}"` is on the value span for the TileHero morph.

12 metrics: `sleep_duration`, `sleep_efficiency`, `sleep_latency`, `hrv_rmssd`, `rhr`, `hr_avg`, `stress_avg`, `spo2_avg`, `skin_temp_delta`, `steps`, `active_minutes`, `breath_rate`. The last three (`skin_temp_delta`, `breath_rate`, plus apnea event count) are the new signals from issue #44.

**Year heatmap**: a client component `<YearHeatmap metric="sleep_duration">` rendering a scrollable 53×7 cell grid. Each cell: background color interpolated from the domain accent (e.g. `--color-sleep`) scaled by value quantile. Cell tap → `/today?date=YYYY-MM-DD`. Metric switcher: a pill row above the heatmap (sleep, hrv, steps, stress, spo2) using the `layoutId="heatmap-metric-pill"` sliding indicator. This is the `/explore/heatmap` content, also embedded below the fold on `/explore`.

### `/explore/[metric]` — single-metric deep-dive

**Structure**:
1. Hero: large metric value (TileHero morph from grid), 14-day sparkline, percentile ring (`RingGauge` repurposed with smaller size).
2. Timeline chart (full width): the relevant chart component for the metric (e.g. `HrTimeline` for `hr_avg`, `Hypnogram` for sleep). Anomaly callouts rendered as `ReferenceArea` + `AnomalyRow` directly on the chart — not a separate tab. "Why?" button on each anomaly (inline expand, T3).
3. Distribution panel (half width on desktop): histogram or scatter (e.g. `HrvScatter`, `Spo2Distribution`). Includes percentile band overlay showing user's rolling 30-day band ± 1σ as a shaded `ReferenceArea`.
4. Week overlay (half width on desktop): 7 lines, one per day of the trailing week, same chart type but thin. Allows visual "is today normal vs this week" comparison.
5. Sample browser: paginated list of raw readings. Each row: timestamp, value, context (sleep/awake flag). Rows are expandable (FLIP, T3). Accessible via `[raw data ↗]` link.

Date range: defaults to the last available day. `?date=` param anchors to a specific day. The user can also drag-select a time window on the timeline chart (T8) to zoom in-chart; this does not change the route.

### `/explore/heatmap`

Full-screen version of the year heatmap. Metric switcher above. Cell click → `/today?date=`. CSR only — no RSC data needed beyond initial metric.

### `/explore/correlate`

Two metric selectors (dropdowns, left and right axis). Date range picker (14d / 30d / 90d / custom). A dual-Y scatter/overlay chart (`ComposedChart` from Recharts). No LLM. Pearson r displayed as a stat chip. This is the Levels-style cross-metric overlay from the UX research catalog.

### `/explore/patterns`

Renders Phase 3 LLM output: named recurring signatures + surprise-ranked top-N insights.

**Named patterns** (PROBE_pattern_naming_and_surprise.md): rendered as cards with pattern name, description, occurrence count, days list as small date chips. The PROBE found salience blindness — the pattern name must be prefixed by the deterministic `salient_flags` pre-ranking. Cards link to `/explore/[metric]?pattern=id` which highlights the relevant days on the timeline.

**Surprise-ranked insights** (PROBE B, NEEDS-ITERATION): rendered as a ranked list with `surprise_label` (hard-rule: |z|≥3 → high, ≥1.5 → medium, else low — computed deterministically per PROBE recommendation). The LLM only generates the headline and reason text. `n_baseline < 5` → `fragile: true` badge shown.

## 8. `/log` Architecture (Phase 4)

### `/log` — index

Timeline of recent entries (weight, journal, feel, screenshot). Each entry: date, type icon, summary value. Tap → detail view (not spec'd in v2.1). RSC, fresh on every request.

### `/log/weight`

Form: weight (kg, 1 decimal), optional body-fat % (if Huawei Health data not available). Submit → server action → writes to `body_comp_samples` table → revalidates `/today` and `/explore`. Client form (`"use client"`) with optimistic update.

### `/log/journal`

Form: free text (textarea), mood (5-point scale via icon row), tags (multiselect chips: `#alcohol`, `#travel`, `#illness`, `#training`, `#stress`). These tags feed directly into the journal context sent to anomaly-explanation and coaching-trajectory prompts.

### `/log/feel`

Minimal: single 1–5 subjective feel slider, optional one-line note. Quick-log, loads fast. Feeds into the `journal_7d` stub in the anomaly explanation scope.

### `/log/screenshot`

Upload UI: drag-drop or file picker for a Huawei Health body-comp screenshot. On select: preview the image, send to `/api/ingest-screenshot` (server action calling Ollama vision, PROBE_vision.md: SHIP decision, long-edge ≤720 px resize before send, ~60 s latency). Response: extracted fields with confidence. User sees a "confirm these values?" review step before commit. Reject if any field confidence ≠ `high` or label not in whitelist.

## 9. Information Density Tradeoffs

**Rule: editorial density on glance routes, data density on session routes.**

| Context | Rule |
|---|---|
| `/today` mobile | Single verdict + 4 domain tiles + 2-3 anomalies visible without scroll. No charts above fold. |
| `/today` desktop | Same content, wider layout. Domain tiles go 1×4 instead of 2×2. |
| `/today/[domain]` | Charts appear. Dense is acceptable here — user chose to drill in. |
| `/explore` mobile | 2-column KPI grid. Year heatmap below fold, scrollable. One chart per `/explore/[metric]`. |
| `/explore` desktop | 4-column KPI grid. `/explore/[metric]` shows timeline + distribution + week overlay simultaneously. |
| `/explore/correlate` | Desktop-first; mobile shows stacked single chart. |
| Anomaly "Why?" expansion | Same width as the row — no layout reflow beyond the row's siblings sliding down. |
| CoachCards | Trajectory + tiny step visible; projection behind `<details>`. Prevents information overload on `/today`. |

**Dense acceptable when**: user navigated to a drill-down route or `/explore`. The user explicitly chose depth.

**Editorial enforced when**: `/today` top-of-fold, `/week` recap. These are narrative surfaces; charts are below fold or behind domain links.

## 10. State, Caching, Freshness

### SSR / RSC / CSR per route

| Route | Rendering | Cache strategy |
|---|---|---|
| `/today` | RSC + SSR | `cache: 'no-store'` — daily data changes when runner runs (~daily). Reads from local JSON files. |
| `/today/[domain]` | RSC + SSR | Same — reads SQLite via synchronous queries in RSC. |
| `/week` | RSC + SSR | `cache: 'no-store'` |
| `/alarms` | RSC + SSR | `cache: 'no-store'` |
| `/explore` | RSC shell + CSR chart tiles | Shell SSR; KPI values computed server-side and serialized as props. Chart CSR. |
| `/explore/[metric]` | RSC for data fetch, CSR for charts | `cache: 'no-store'` for the data; chart rendering CSR. |
| `/explore/heatmap` | Pure CSR | Data fetched client-side from a `/api/heatmap?metric=&year=` route handler. Route handler: `revalidate: 3600` (1 h). |
| `/explore/correlate` | Pure CSR | Client fetches `/api/metric-series` per selection. Route handler: `revalidate: 3600`. |
| `/explore/patterns` | RSC | `revalidate: 3600` — pattern names are stable within a day. |
| `/log/*` | CSR forms | No caching; server actions on submit. |
| `/api/explain-anomaly` | Route handler | `no-store` — on-demand LLM call, ~26 s. |

### Write-back revalidation

Server actions in `/log/*` call `revalidatePath('/today')`, `revalidatePath('/explore')`, and `revalidatePath('/explore/patterns')` after successful writes. The year heatmap is revalidated by `revalidatePath('/explore/heatmap')`. The `revalidatePath` calls are cheap because the underlying data is SQLite local reads.

### LLM response caching

Anomaly explanations are expensive (~26 s) and deterministic for a given `(anomaly_id, date, 7-day window)`. Cache in a `explanations/` JSON sidecar keyed by `{observation_id}_{period_key}.json`. Check sidecar before hitting Ollama. Cache is file-system only (same pattern as existing `_facts.json` / `daily/*.json` approach).

Coaching trajectory cards are cached per `(lever_id, iso_week)` to prevent tiny-step churn (per the coaching-trajectory probe finding).

## 11. Decision Log

1. **Default route = `/today`**: the daily LLM article is the primary value proposition — a quick emotional check-in. `/explore` is for deliberate sessions, not daily opens. Switching the default would punish every morning glance with a chart-grid requiring orientation. Decision: `/today` stays default.

2. **Explore as a new top-level route, not a sub-route of `/today`**: Explore is a different mode of engagement (session vs. glance). Putting it under `/today/explore` would bury it and violate the breadcrumb hierarchy. It gets its own nav slot.

3. **"Why?" expands inline, not modal**: modals break scroll context and hide the chart the anomaly lives on. The user wants to see the anomaly callout and its hypotheses simultaneously. Inline FLIP expansion (T3) keeps spatial context. Precedent: `ApneaEventRow` already does this pattern (`motion.li layout` + `motion.div height 0 → auto`).

4. **LLM anomaly explanation: Probe 2 scope, on-demand, cached**: Probe 2 (7-day `_facts.json` window) is the correct scope per PROBE_anomaly_explanation.md — it produces real day-over-day deltas. On-demand (not pre-computed) because not all anomalies will be tapped. Cached to avoid re-invoking Ollama on repeat views.

5. **Pattern naming requires pre-rank + `salient_flags`**: PROBE_pattern_naming_and_surprise.md found qwen3.6 picks a coherent narrative, not the salient one — safety-relevant miss on the tachycardia cluster. Deterministic pre-ranking must inject `salient_flags` before the LLM call. Pattern naming is therefore NEEDS-ITERATION and ships without the naming LLM until the re-probe confirms the fix.

6. **Surprise labels computed deterministically**: PROBE B found label drift for the same z-score across runs. Hard rules: |z|≥3 → high, ≥1.5 → medium, else low. LLM only generates the headline + reason. This is the recommended fix from the PROBE.

7. **Year heatmap as navigation primitive, not decoration**: the GitHub-grid heatmap is the best-in-class long-horizon navigation for a data person. Every cell is a link to `/today?date=`. It appears on both `/explore` (below fold) and `/explore/heatmap` (full screen). This makes the heatmap a spatial index into the dataset.

8. **Woche demoted from primary nav to sheet on mobile**: the four-item mobile nav needs Explore more than it needs Woche. Week is a session-frequency feature (checked ~weekly); Explore is a daily session feature. Woche moves to the "More" sheet alongside Profil/Labs.

9. **No persistent date state in URL for `/explore` grid**: the metric grid landing `/explore` is stateless by default. Date context only enters the URL when drilling to a specific metric with `?date=`. This keeps the grid shareable as a stable landing.

10. **CoachCards on `/today`, not only on `/week`**: coaching trajectory (PROBE: SHIP) is per-lever and daily. Relegating it to `/week` delays the feedback loop. Each active lever gets one card on `/today` below the anomalies section, collapsed by default on mobile to preserve editorial density.

11. **Vision ingest as manual flow, not automatic**: PROBE_vision.md: SHIP for Huawei body-comp. But the latency is ~60 s and confidence calibration is not yet hardened. The flow is user-initiated from `/log/screenshot`, not automated. This preserves data integrity and gives the user a confirm step.

12. **Remove reasoning trace from `/today` production UI**: the `<details>` reasoning trace is a debug artefact. It adds visual weight to a surface that should feel like a clean editorial read. Move behind a dev flag or to `/labs`.

Should the heatmap load a single metric at a time (current proposal: yes, metric switcher changes the fetch), and should it cache aggressively (`revalidate: 86400`)? User confirmation needed to set cache horizon.
